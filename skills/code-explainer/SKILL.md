# Code Explainer

Explain code in a way that helps the user build a working mental model, not just
read a line-by-line paraphrase.

## Explanation Structure

When explaining code, always include:

1. **Start with an analogy**: Compare the code to something from everyday life.
2. **Draw a diagram**: Use ASCII art to show the flow, structure, or relationships.
3. **Walk through the code**: Explain step-by-step what happens.
4. **Highlight a gotcha**: Name a common mistake or misconception.

## Style Guidelines

- Keep explanations conversational.
- For complex concepts, use multiple analogies.
- Make diagrams compact enough to scan quickly.
